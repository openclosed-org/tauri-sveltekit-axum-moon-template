# Phase 1, Plan 03: Tauri Plugin Registration

## Objective

Add missing Tauri plugins to `apps/desktop-ui/src-tauri/Cargo.toml` so all 7 plugins declared in the root workspace are available to the Tauri app.

**Purpose:** The Tauri app's Cargo.toml must explicitly list each plugin it uses. Currently only 3 of 7 required plugins are declared. Without this, `tauri::Builder::plugin()` calls will fail to compile.

**Output:** Updated `apps/desktop-ui/src-tauri/Cargo.toml` with all 7 Tauri plugins referenced via workspace.

## Context

Current `apps/desktop-ui/src-tauri/Cargo.toml` state:
- Has 3 plugins: `tauri-plugin-shell`, `tauri-plugin-dialog`, `tauri-plugin-store` ✅
- **Missing 4 plugins:** `tauri-plugin-fs`, `tauri-plugin-deep-link`, `tauri-plugin-window-state`, `tauri-plugin-libsql`
- All missing plugins will be available in root workspace after Plan 02 completes

Per TECH_SELECTION.md §5.2:
- `tauri-plugin-fs` — File system access with permissions (Phase 8 desktop features)
- `tauri-plugin-deep-link` — OAuth callback handling (Phase 6 auth)
- `tauri-plugin-window-state` — Window position/size persistence (Phase 8)
- `tauri-plugin-libsql` — Local embedded database (Phase 5)

These are "preload" declarations — the plugins are declared now but will be used in later phases.
