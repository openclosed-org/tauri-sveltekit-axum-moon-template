  <files>.agents/rubrics/code-review.md</files>
  <action>
创建 .agents/rubrics/code-review.md，采用与 boundary-compliance.md 一致的格式。

文件结构必须包含：

1. **Header**：
   - Purpose: Agent code review checklist for code quality (complements boundary-compliance.md)
   - Enforced by: CI (cargo test, vitest, tsc) + agent review (this rubric)

2. **Naming Conventions（命名规范）**：
   - Rust: snake_case 函数/变量，PascalCase 类型/traits，SCREAMING_SNAKE_CASE 常量
   - TypeScript: camelCase 函数/变量，PascalCase 组件/接口，kebab-case 文件名
   - 禁止：单字母变量名（除循环计数器 i/j/k）、缩写歧义（如 `ctx` 应写全称 `context`）
   - 文件命名：Rust 用 snake_case.rs，Svelte 用 PascalCase.svelte

3. **Error Handling（错误处理）**：
   - Rust: 使用 thiserror 定义错误类型，不使用 unwrap() 除测试代码外
   - TypeScript: 使用 try/catch 包裹异步操作，不吞异常
   - API: 返回结构化错误响应（{error: {code, message, details}} 格式）
   - 禁止：console.log 留在生产代码、空 catch 块、panic! 在非测试代码

4. **Test Coverage（测试覆盖）**：
   - Rust: 新增函数必须有 #[cfg(test)] 单元测试
   - Svelte: 新增组件必须有 Vitest 渲染测试
   - API: 新增端点必须有集成测试
   - 覆盖率要求：新增代码行覆盖率 ≥ 80%
   - 测试命名：describe/it 格式，描述行为而非实现

5. **Documentation Completeness（文档完整性）**：
   - 新增公开函数必须有 /// 文档注释（Rust）或 JSDoc（TypeScript）
   - 新增 feature 必须更新 AGENTS.md 或相关 README
   - 新增 API 端点必须更新 OpenAPI spec（utoipa 注解）
   - 破坏性变更必须包含迁移说明

6. **Review Checklist（审查清单）** — checkbox 格式：
   - [ ] 命名符合约定，无歧义缩写
   - [ ] 错误处理完整，无 unwrap/panic/console.log
   - [ ] 新增代码有对应测试
   - [ ] 公开函数有文档注释
   - [ ] 无跨层违规 import（见 boundary-compliance.md）
   - [ ] 无未使用的变量/导入/依赖
   - [ ] 函数长度 ≤ 50 行，文件长度 ≤ 800 行
   - [ ] 嵌套深度 ≤ 4 层

格式参照 .agents/rubrics/boundary-compliance.md（H2/H3 标题 + checkbox 列表 + MUST/CAN 语言）。

<task type="auto">
  <name>Task 2: 创建 task-completion rubric</name>
  <files>.agents/rubrics/task-completion.md</files>
  <action>
创建 .agents/rubrics/task-completion.md，定义任务完成的验收标准。

文件结构必须包含：

1. **Header**：
   - Purpose: Checklist to verify a task/feature is fully complete before marking done
   - Enforced by: Agent self-check + CI (just verify)

2. **Requirements Alignment（需求对齐）**：
   - 任务目标已明确且可验证
   - 所有 acceptance criteria 已满足
   - 输入/输出/约束已确认
   - 不确定项已标注并说明影响

3. **Verification Passed（验证通过）**：
   - `just verify` 全量检查通过
   - `just typegen` 无漂移
   - `cargo test` 全部通过
   - 前端测试（Vitest/Playwright）全部通过
   - 无 lint 警告或已确认忽略

4. **No Regression（无回归）**：
   - 现有功能未受影响（运行完整测试套件）
   - 未引入新的编译警告
   - 未修改无关文件或目录结构
   - 未引入新依赖（除非明确批准）
   - 二进制体积无显著增长（> 1MB 需说明原因）

5. **Documentation Updated（文档更新）**：
   - AGENTS.md 如有相关规则变更已更新
   - README.md 如有使用方式变化已更新
   - API 文档（OpenAPI/utoipa）已同步
   - 变更说明（CHANGELOG 或 commit message）已包含
   - 如影响其他模块，已通知相关维护者

6. **Output Format（输出格式）** — 完成时必须输出：
   - 任务目标（一句话）
   - 改动摘要（改了什么、为什么）
   - 涉及文件列表
   - 验证结果（哪些检查通过）
   - 风险/限制/未完成项
   - 下一步建议（如有）

7. **Completion Checklist（完成清单）** — checkbox 格式：
   - [ ] 需求对齐：acceptance criteria 全部满足
   - [ ] 验证通过：just verify / cargo test / vitest 全部通过
   - [ ] 无回归：现有测试全部通过，无新警告
   - [ ] 文档更新：相关文档已同步
   - [ ] 输出格式：按 AGENTS.md 第 6 节格式输出总结
   - [ ] 边界合规：无跨层违规 import（见 boundary-compliance.md）
   - [ ] 代码质量：通过 code-review rubric 检查

格式参照 .agents/rubrics/boundary-compliance.md。
