# Phase 1, Plan 02: Rust Workspace Dependencies

## Objective

Configure root `Cargo.toml` with all workspace dependencies pinned to TECH_SELECTION.md versions, including Tauri plugins, backend stack, and release profile optimization.

**Purpose:** Root Cargo.toml is the version authority for the entire Rust workspace. All crate Cargo.toml files reference these via `{ workspace = true }`. Pinning versions here prevents dependency drift and ensures consistent builds across the monorepo.

**Output:** Updated root `Cargo.toml` with complete workspace dependency declarations and optimized release profile.

## Context

Current root `Cargo.toml` state:
- Has 5 workspace dependencies: `tauri "2"`, `tauri-build "2"`, `tauri-plugin-shell "2"`, `tauri-plugin-dialog "2"`, `tauri-plugin-store "2"`
- **Missing 4 plugins:** `tauri-plugin-fs`, `tauri-plugin-deep-link`, `tauri-plugin-window-state`, `tauri-plugin-libsql`
- **Missing all backend deps:** `axum`, `tokio`, `reqwest`, `surrealdb`, `serde`, `serde_json`, `jsonwebtoken`, `uuid`, `chrono`
- **Loose versions:** `"2"` instead of pinned `"2.10.3"` etc.
- **No release profile:** No `[profile.release]` section for binary optimization

Per TECH_SELECTION.md §5.2, §7.1 (已验证 2026-03-28):
- Tauri plugins: shell, dialog, store, fs, deep-link, window-state, libsql
- Backend: axum 0.8.8, tokio 1.50.0, reqwest 0.13.2, surrealdb 3.0.5, serde 1.0.228, jsonwebtoken 10.3.0, uuid 1.23.0, chrono 0.4

Per RESEARCH.md (Pitfall 2): Binary size optimization must be configured from the start — LTO, codegen-units=1, opt-level="z", strip=true.
