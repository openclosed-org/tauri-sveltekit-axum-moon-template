## 跨问题根因分析

**根因 A — 无 Web/Desktop 运行时抽象层：**
- 多处文件顶层 import `@tauri-apps/api/*`，在浏览器环境下直接崩溃
- 涉及文件：`+layout.svelte`、`auth.ts`、`agent.ts`、`auth.svelte.ts`
- 需要统一的 `isTauri` 检测 + 条件 import 或 safe wrapper

**根因 B — API 路由前缀不一致：**
- Counter/Tenant 路由：有 `/api` 前缀
- Admin/Agent 路由：无 `/api` 前缀
- 前端统一使用 `/api/...` 前缀
- 需要统一：所有路由加 `/api` 前缀，或从 Router 层统一 mount

**根因 C — Web 模式无持久化策略：**
- Settings：Tauri Store only，web 模式无 fallback
- Auth token：web 模式无存储/读取机制
- 需要 localStorage fallback 或后端持久化

## 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| Web/Desktop 运行时检测方式 | `safeInvoke`/`safeListen` wrapper 模块 | 集中管理，避免每个文件重复检测；可返回合理 fallback |
| Settings web 持久化 | localStorage | 最小改动，无需后端 API 变更 |
| Auth token web 持久化 | localStorage | 与 Settings 一致；后续可升级为 httpOnly cookie |
| 路由前缀统一 | 后端所有路由加 `/api` 前缀 | 前端已统一使用 `/api`，改后端更简单 |
| Agent Chat 闪烁修复 | 移除 finally 中全量重取，改用增量更新 | 减少不必要的网络请求和 re-render |
