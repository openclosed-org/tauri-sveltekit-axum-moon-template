# PLAN 10.3-03: Settings 持久化 + Agent Chat 闪烁修复 + 对话排序

**Goal:** Web 模式下 Settings 可持久化，Agent Chat 发送消息无闪烁，对话列表排序稳定
**Depends on:** 10.3-01, 10.3-02
**Requirements:** UAT-03, UAT-04, UAT-06
**Verification:** web 模式保存 settings 后刷新页面仍存在，agent 发送消息无闪烁，对话列表顺序稳定

## Context

Web 模式下 Settings 无持久化 (Tauri Store only)，Agent Chat 的 finally 块全量重取导致闪烁，对话列表无排序导致跳闪。

## Task 1: Settings 页面 localStorage fallback

**File:** `apps/client/web/app/src/routes/(app)/settings/+page.svelte:39-78`

修改 `loadSettings()` — 在 else 分支添加 localStorage fallback：
```javascript
} else {
  // Web mode: localStorage fallback
  apiKey = localStorage.getItem('settings_api_key') ?? '';
  baseUrl = localStorage.getItem('settings_base_url') ?? 'https://api.openai.com/v1';
  model = localStorage.getItem('settings_model') ?? 'gpt-4o-mini';
}
```

修改 `saveSettings()` — 在 else 分支添加 localStorage persistence：
```javascript
} else {
  // Web mode: localStorage fallback
  localStorage.setItem('settings_api_key', apiKey);
  localStorage.setItem('settings_base_url', baseUrl);
  localStorage.setItem('settings_model', model);
}
```

## Task 2: Agent Chat 页面 Settings fallback

**File:** `apps/client/web/app/src/routes/(app)/agent/+page.svelte:24-49`

修改 `loadSettings()` 函数，在 web 模式 else 分支添加 localStorage fallback，返回从 localStorage 读取的 settings。

## Task 3: Agent Chat 发送消息 — 移除 finally 全量重取

**File:** `apps/client/web/app/src/routes/(app)/agent/+page.svelte:175-181`

```javascript
// Before
} finally {
    streaming = false;
    await loadConversations();
    if (activeConversation) {
        await selectConversation(activeConversation);
    }
}

// After
} finally {
    streaming = false;
}
```

理由：in-memory messages 已经是最新的 (optimistic update + streaming append)，无需重新获取。

## Task 4: Auth token web 模式持久化

**File:** `apps/client/web/app/src/lib/ipc/auth.ts` (需在 10.3-01 bridge 改造后)

在 web 模式下：
- `setSession` 时：`localStorage.setItem('auth_id_token', session.id_token)`
- `clearAuthStore` 时：`localStorage.removeItem('auth_id_token')`
- `getSession` 时：从 localStorage 读取并返回

这确保 counter 页面 (Task 3 of 10.3-02) 能读取到 token。

## Task 5: 对话列表排序

**File:** `apps/client/web/app/src/routes/(app)/agent/+page.svelte:52-68`

在 `loadConversations()` 中 fetch 后排序：
```javascript
const fetched = await listConversations();
conversations = fetched.sort((a, b) => {
  const dateA = a.created_at ?? '';
  const dateB = b.created_at ?? '';
  return dateB.localeCompare(dateA);
});
```

## Verification

1. Web 模式 Settings：
   - 输入 API Key -> Save -> 刷新页面 -> API Key 仍存在
   - 切换到 Counter 再切回 Settings -> API Key 仍存在
2. Web 模式 Agent Chat：
   - 创建新对话 -> 无跳闪排序
   - 发送消息 -> 页面无闪烁，消息正常显示
3. Web 模式 Counter：
   - 点击加号 -> 数值从 0 变为 1 (无 "failed to increment")
4. Tauri 桌面模式：
   - 所有功能不受影响 (Settings 仍使用 Tauri Store)
5. `bun run check` — 无类型错误
