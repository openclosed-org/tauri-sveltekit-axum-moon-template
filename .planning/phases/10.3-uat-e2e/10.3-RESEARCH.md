## 1. UAT-01: Counter 增减/重置失败

### Root Cause
**File:** `apps/client/web/app/src/routes/(app)/counter/+page.svelte:23`

Frontend `invokeCommand()` 在 web 模式下做 fetch 调用时，未携带 `Authorization: Bearer <token>` header：

```javascript
const resp = await fetch(url, { method });  // 无 auth header
```

后端 counter 路由在 `tenant_middleware` 保护下（`servers/api/src/lib.rs:81-82`），middleware 要求 Bearer token（`servers/api/src/middleware/tenant.rs:32-38`），缺失则返回 401。

### Fix
- Counter 页面需要获取当前 auth token 并加入 fetch headers
- Token 来源：`auth` store 中的 session 信息，或 localStorage 中存储的 id_token
- 需要与 UAT-03（Settings persistence）联动：web 模式下 auth token 需要持久化才能被 counter 页面读取

### Affected Files
- `apps/client/web/app/src/routes/(app)/counter/+page.svelte:16-26` — invokeCommand 函数

## 2. UAT-02: Web 端 Google 登录崩溃

### Root Cause
多处顶层 import `@tauri-apps/api/*`，在浏览器环境下 `window.__TAURI_INTERNALS__` 为 undefined，调用时直接崩溃。

**关键文件与行号：**

| File | Line | Import |
|------|------|--------|
| `apps/client/web/app/src/routes/+layout.svelte` | 2-3 | `invoke`, `listen` |
| `apps/client/web/app/src/lib/ipc/auth.ts` | 3 | `invoke` |
| `apps/client/web/app/src/lib/ipc/agent.ts` | 2 | `Channel`, `invoke` |
| `apps/client/web/app/src/lib/stores/auth.svelte.ts` | 4 | `listen` |

**崩溃链：**
1. `+layout.svelte:2` 顶层 import `invoke` from `@tauri-apps/api/core`
2. `+layout.svelte:31` onMount 调用 `initAuthListeners()`
3. `auth.svelte.ts:145` `initAuthListeners()` 调用 `listen('auth:expired', ...)`
4. `listen` 内部调用 `invoke('plugin:event|listen', ...)`
5. `invoke` 访问 `window.__TAURI_INTERNALS__.invoke` → undefined → crash

### Fix 方案
创建 `$lib/ipc/bridge.ts` wrapper 模块，集中管理运行时检测：
- `isTauri()` — 检测 `window.__TAURI__`
- `safeInvoke()` — Tauri 模式调用 invoke，web 模式返回合理 fallback 或抛出明确错误
- `safeListen()` — Tauri 模式调用 listen，web 模式返回 no-op cleanup 函数

各文件改为 import bridge wrapper 而非直接 import `@tauri-apps/api/*`。

### Affected Files
- `apps/client/web/app/src/routes/+layout.svelte:2-3,29-70`
- `apps/client/web/app/src/lib/ipc/auth.ts:3,16-32`
- `apps/client/web/app/src/lib/ipc/agent.ts:2,43,58,78,114,126`
- `apps/client/web/app/src/lib/stores/auth.svelte.ts:4,144-154`
- 新建 `apps/client/web/app/src/lib/ipc/bridge.ts`

## 3. UAT-03: Settings 保存丢失

### Root Cause
**File:** `apps/client/web/app/src/routes/(app)/settings/+page.svelte:39-51,54-78`

`loadSettings()` 和 `saveSettings()` 仅在 `window.__TAURI__` 存在时执行 Tauri Store 操作。Web 模式下：
- `loadSettings()`: if 块跳过，变量保持硬编码默认值
- `saveSettings()`: if 块跳过，`saved = true` 但无实际持久化

同样问题存在于 Agent Chat 页面（`agent/+page.svelte:24-49`）。

### Fix
Web 模式 fallback 到 localStorage：
- `loadSettings()`: `localStorage.getItem('settings_api_key')` 等
- `saveSettings()`: `localStorage.setItem('settings_api_key', apiKey)` 等

### Affected Files
- `apps/client/web/app/src/routes/(app)/settings/+page.svelte:39-78`
- `apps/client/web/app/src/routes/(app)/agent/+page.svelte:24-49`

## 4. UAT-04: Agent Chat 发送消息页面闪烁

### Root Cause A — finally 块全量重取
**File:** `apps/client/web/app/src/routes/(app)/agent/+page.svelte:175-181`

```javascript
} finally {
    streaming = false;
    await loadConversations();           // 全量重取会话列表
    if (activeConversation) {
        await selectConversation(activeConversation);  // 全量重取消息
    }
}
```

`selectConversation` 会替换整个 `messages` 数组（line 89）。若后端尚未持久化 streaming 消息，重取结果可能为空，导致 UI 闪烁。

### Root Cause B — 路由前缀不匹配
**File:** `apps/client/web/app/src/lib/ipc/agent.ts:51,66,86,167`

前端 HTTP fallback 请求 `/api/agent/conversations` 和 `/api/agent/chat`。

**File:** `servers/api/src/routes/agent.rs:19,23`

后端路由定义为 `/agent/conversations` 和 `/agent/chat`（无 `/api` 前缀）。

Web 模式下所有 agent API 调用返回 404。

### Fix
- 移除 finally 中的全量重取，改为仅更新 `streaming = false`
- 后端 agent 路由加 `/api` 前缀：`/api/agent/conversations`、`/api/agent/chat`

### Affected Files
- `apps/client/web/app/src/routes/(app)/agent/+page.svelte:175-181`
- `servers/api/src/routes/agent.rs:19,22,23`

## 5. UAT-05: Admin Dashboard 白色主题下数字不可见

### Root Cause A — 路由前缀不匹配
**File:** `apps/client/web/app/src/routes/(app)/admin/+page.svelte:26`

前端请求 `http://localhost:3001/api/admin/stats`

**File:** `servers/api/src/routes/admin.rs:8`

后端路由定义为 `/admin/stats`（无 `/api` 前缀）→ 404，数据不加载

### Root Cause B — CSS 变量问题
Admin 页面使用 `text-[var(--color-text)]` 和 `text-[var(--color-text-muted)]`。

CSS 变量定义在 `app.css:41-48`：
```css
:root {
  --color-text: var(--color-gray-900);
  --color-text-muted: var(--color-gray-500);
}
```

Tailwind v4 中 `text-[var(--color-text)]` 可能不被正确处理为 CSS 值，导致 text color 未生效，继承浏览器默认（可能是白色背景下的白色）。

### Fix
- 后端 admin 路由加 `/api` 前缀：`/api/admin/stats`
- CSS：将 `text-[var(--color-text)]` 替换为 Tailwind 主题 token `text-gray-900 dark:text-gray-50`（与 `@theme` 定义对齐）

### Affected Files
- `servers/api/src/routes/admin.rs:8`
- `apps/client/web/app/src/routes/(app)/admin/+page.svelte:53,54,60,63`

## 6. UAT-06: New Chat 标签跳闪排序

### Root Cause
**File:** `apps/client/web/app/src/routes/(app)/agent/+page.svelte:52-68,70-83`

创建新对话后调用 `loadConversations()` 全量重取。后端 `get_conversations()` 无 `ORDER BY`，SQLite 默认返回顺序不稳定，导致 sidebar 渲染时项目位置跳变。

### Fix
- 后端查询加 `ORDER BY created_at DESC`
- 或前端 fetch 后排序：`conversations.sort((a, b) => b.created_at - a.created_at)`

### Affected Files
- `servers/api/src/routes/agent.rs:53-63` (或对应的 usecases agent_service)
- `apps/client/web/app/src/routes/(app)/agent/+page.svelte:52-68`
