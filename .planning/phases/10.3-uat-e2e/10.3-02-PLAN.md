# PLAN 10.3-02: API 路由前缀统一 + Counter auth header

**Goal:** 修复 admin/agent 404 和 counter 401，使 web 模式下 API 调用正常工作
**Depends on:** 10.3-01
**Requirements:** UAT-01, UAT-04, UAT-05
**Verification:** web 模式下 counter 增减/重置正常，admin 数据加载正常，agent 会话列表正常

## Context

后端 admin 和 agent 路由缺少 `/api` 前缀，与前端 URL 不匹配导致 404。Counter 路由前缀正确但 fetch 缺少 Authorization header 导致 401。

**File:** `servers/api/src/routes/admin.rs:8`

```rust

// After

```

## Task 3: Counter fetch 添加 Authorization header

**File:** `apps/client/web/app/src/routes/(app)/counter/+page.svelte:16-26`

修改 `invokeCommand()` 函数：从 localStorage 读取 auth token，fetch 调用时加入 `Authorization: Bearer <token>` header。

```javascript
async function invokeCommand(cmd: string) {
  if (tauriApi) {
    return tauriApi.core.invoke(cmd);
  }
  const method = cmd.startsWith('counter_get') ? 'GET' : 'POST';
  const endpoint = cmd.replace('counter_', '').replace('get_value', 'value');
  const url = `http://localhost:3001/api/counter/${endpoint}`;
  const token = localStorage.getItem('auth_id_token');
  const headers: Record<string, string> = {};
  if (token) {
    headers['Authorization'] = `Bearer ${token}`;
  }
  const resp = await fetch(url, { method, headers });
  const data = await resp.json();
  return data.value;
}
```
