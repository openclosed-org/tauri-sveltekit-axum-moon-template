# PLAN 10.3-01: Web/Desktop 运行时抽象层 + Tauri import 修复

**Goal:** 消除 `window.__TAURI_INTERNALS__` 崩溃，使 web 模式可正常启动
**Depends on:** none
**Requirements:** UAT-02
**Verification:** 浏览器访问 http://localhost:5173/ 不崩溃，页面正常渲染

## Context

多处文件顶层 import `@tauri-apps/api/*`，在浏览器环境下直接崩溃。需要集中管理运行时检测，避免每个文件重复判断。

## Task 1: 创建 `$lib/ipc/bridge.ts`

新建 `apps/client/web/app/src/lib/ipc/bridge.ts`，导出：

- `isTauri(): boolean` — 检测 `window.__TAURI__` 是否存在
- `safeInvoke(cmd, args?)` — Tauri 模式调用 `invoke`，web 模式抛出明确错误
- `safeListen(event, callback)` — Tauri 模式调用 `listen`，web 模式返回 no-op cleanup 函数
- `safeChannel<T>()` — Tauri 模式创建 Channel，web 模式返回 null

**关键约束：**
- 不能顶层 import `@tauri-apps/api/*`，必须用 dynamic import（`await import(...)`）
- `isTauri()` 检查 `typeof window !== 'undefined' && !!window.__TAURI__`

## Task 2: 修改 `+layout.svelte`

**File:** `apps/client/web/app/src/routes/+layout.svelte:2-3,29-70`

- 改为 import `safeListen`, `isTauri` from `$lib/ipc/bridge`
- `onMount` 中：

  - `invoke('quit_app')` 改为 guarded：仅在 `isTauri()` 时调用
