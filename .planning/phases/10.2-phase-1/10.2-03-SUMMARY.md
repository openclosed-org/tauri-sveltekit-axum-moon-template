# Phase 10.2 Plan 03: AgentConfig api_key 序列化脱敏与 Debug 脱敏 Summary

## One-liner

Redacted `api_key` from `AgentConfig` serialization (serde skip_serializing) and Debug output (manual impl with `[REDACTED]`).

## What Was Done

1. **Serialization redaction:** Added `#[serde(skip_serializing)]` on `AgentConfig.api_key` — field is omitted from JSON output but remains deserializable for incoming requests.
2. **Debug redaction:** Replaced `#[derive(Debug)]` with a manual `fmt::Debug` impl that shows `[REDACTED]` instead of the actual api_key value.
3. **Tests added:** Three new tests verifying serialize redaction, deserialization correctness, and Debug output redaction.

## Key Changes

### `packages/contracts/api/src/lib.rs`

- Added `use std::fmt;`
- Changed `#[derive(Debug, Clone, Serialize, Deserialize, ToSchema, TS)]` → `#[derive(Clone, Serialize, Deserialize, ToSchema, TS)]`
- Added `#[serde(skip_serializing)]` on `api_key` field
- Added manual `impl fmt::Debug for AgentConfig` that redacts api_key as `[REDACTED]`
- Added 3 tests: `agent_config_serialize_redacts_api_key`, `agent_config_deserialize_populates_api_key`, `agent_config_debug_redacts_api_key`

## Decisions Made

**Skipped `#[ts(skip)]`:** The RESEARCH.md suggested adding `#[ts(skip)]` to remove `api_key` from the generated TypeScript type. However, the existing frontend code (`apps/client/web/app/src/routes/(app)/agent/+page.svelte`) imports the generated `AgentConfig` type and uses `api_key` for local settings management (reading from Tauri Store). Adding `#[ts(skip)]` would break this code. Since `api_key` is never returned in API responses (due to `skip_serializing`), keeping it in the TS type is safe and avoids unnecessary frontend changes.

## Deviations from Plan

None — plan executed exactly as written (plus the `#[ts(skip)]` omission documented above).
