## 编译裁剪边界

| Option | Description | Selected |
|--------|-------------|----------|
| 默认仅 Turso | 默认开发/测试/CI 只走 Turso 相关编译链路；非 Turso adapter 仅在显式 feature/job 下启用。 | ✓ |
| 默认 Turso+兼容烟测 | 默认走 Turso，但保留一个轻量 Surreal smoke 在常规流水线中持续编译。 | |
| 维持全量默认编译 | 继续保持多存储默认一起编译，靠测试选择控制。 | |

**User's choice:** 默认仅 Turso
**Notes:** 锁定默认门禁只编译 Turso 路径。

| Option | Description | Selected |
|--------|-------------|----------|
| 可选feature且默认关闭 | 保留扩展位，但主二进制与默认任务不直接依赖；需显式 `--features` 才参与编译与测试。 | ✓ |
| 先保留代码但从默认依赖图摘除 | 短期只改 runtime 依赖与 CI 任务，不立即统一 workspace feature 架构。 | |
| 临时冻结到独立分支/目录 | 本阶段几乎不触及主干，但后续恢复成本更高。 | |

**User's choice:** 可选feature且默认关闭
**Notes:** `storage_surreal` / `storage_libsql` 继续保留，但不再进入默认依赖图。

## Turso切换口径

| Option | Description | Selected |
|--------|-------------|----------|
| 默认禁用memory | `runtime_server` 与 desktop 默认都必须文件库；`:memory:` 仅 `#[cfg(test)]` 或测试fixture可用。 | ✓ |
| 默认文件+可选memory | 默认文件库，但保留开发模式可切回 memory 的开关。 | |
| 仅生产禁memory | 仅生产禁 memory，开发默认仍可 memory。 | |

**User's choice:** 默认禁用memory
**Notes:** 当前 `servers/api/src/state.rs` 的 memory 路径与此决策冲突，需要在实现中收敛。

| Option | Description | Selected |
|--------|-------------|----------|
| 失败即终止 | 对路径创建失败/连接失败/迁移失败直接启动失败；不回退到 memory。 | ✓ |
| 失败回退memory | 连接失败时回退 memory 并告警。 | |
| 降级继续运行 | 连接失败时跳过相关能力继续启动。 | |

**User's choice:** 失败即终止
**Notes:** 明确禁止“伪通过”式降级。

## 默认存储切换深度

| Option | Description | Selected |
|--------|-------------|----------|
| 彻底 Turso-only | 默认 runtime/默认命令/默认CI 不再走 Surreal 路径；Surreal 只作为未来显式feature扩展位。 | ✓ |
| 双实现并存但默认 Turso | 代码里保留可直接切换 Surreal provider，只是不作为默认。 | |
| 保持现状只改测试门禁 | 运行态先不动，仅靠测试覆盖 Turso 路径。 | |

**User's choice:** 彻底 Turso-only
**Notes:** 默认运行面与默认命令一起收敛，不只是测试层面切换。

| Option | Description | Selected |
|--------|-------------|----------|
| 统一落 Turso | 租户域与业务域同库/同存储方案，减少分裂语义与验证歧义。 | ✓ |
| 租户域暂留现实现 | 只迁业务表，tenant 域后续再迁。 | |
| 按模块拆分 | 部分租户能力先保留 Surreal，验收口径复杂。 | |

**User's choice:** 统一落 Turso
**Notes:** `tenant` / `user_tenant` 也属于本阶段迁移范围。

## 路径策略

| Option | Description | Selected |
|--------|-------------|----------|
| 平台标准目录 | Tauri 业务库默认放应用数据目录；server 默认放 `servers/api` 下固定文件；均支持 env 覆盖。 | ✓ |
| 统一仓库相对路径 | 两端都放仓库相对路径，便于本地调试。 | |
| 仅环境变量指定 | 完全依赖环境变量，不设默认路径。 | |

**User's choice:** 平台标准目录
**Notes:** 路径既要可用默认值，也要保留 env override。

| Option | Description | Selected |
|--------|-------------|----------|
| 打印完整路径来源 | 启动日志打印 provider + 绝对生效路径 + 来源（default/env），但不打印敏感 token。 | ✓ |
| 仅打印provider | 仅打印 provider，不打印路径。 | |
| 仅debug打印 | 仅在 debug 模式打印路径。 | |

**User's choice:** 打印完整路径来源
**Notes:** 把“路径最终落点可审计”作为本阶段运行态要求。
