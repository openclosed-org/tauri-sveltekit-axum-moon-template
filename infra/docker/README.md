# Docker / Podman

Container configurations and multi-stage builds. Fully compatible with **Podman** (daemonless, rootless).

## Structure

```
docker/
├── compose/
│   ├── app.yaml              # Main application stack (Podman Compose)
│   ├── auth.yaml             # Local auth stack (Rauthy + OpenFGA)
│   └── observability.yaml    # OpenObserve + OTel Collector + optional Vector logs profile
├── compose.dev.yml           # DEPRECATED — legacy Docker Compose
├── Dockerfile.api            # Axum API (distroless/static runtime)
├── Dockerfile.web            # SvelteKit web (static-web-server, Rust)
├── Dockerfile.gateway        # Pingora gateway + static-web-server
└── docker-entrypoint-gateway.sh  # Container-only entrypoint (starts both processes)
```

## Quick Start

```bash
# Start local core infrastructure
cargo run -p repo-tools -- infra local up

# Optional local auth lane: Generic OIDC reference provider + OpenFGA
cargo run -p repo-tools -- infra auth bootstrap

# Or use just commands
just deploy-dev
just status-dev
```

## Stack Decisions

| Old | New | Reason |
|-----|-----|--------|
| Docker | Podman | Daemonless, rootless, systemd-integrated |
| nginx | Pingora + static-web-server | Rust-native, better performance, type safety |
| PostgreSQL | libSQL / optional SurrealDB sidecar | The default backend path uses service-owned libSQL adapters; sidecars are profile-driven |
| Redis | Moka / optional Valkey | In-process cache stays the default backend-core path; Valkey is local/cluster infrastructure when needed |
| NATS server | in-process bus / optional NATS | Single-process flows can use in-process messaging; worker and cluster profiles can use NATS |

## Local Auth Lane

`infra/docker/compose/auth.yaml` is the optional backend auth lane. It currently runs Rauthy as the local reference OIDC provider and OpenFGA as the authorization service.

Runtime-facing env generated by `repo-tools infra auth bootstrap` stays provider-neutral:

1. `APP_OIDC_ISSUER`
2. `APP_OIDC_AUDIENCE`
3. `APP_OIDC_INTROSPECTION_URL`
4. `APP_AUTHZ_PROVIDER`
5. `APP_AUTHZ_ENDPOINT`
6. `APP_AUTHZ_STORE_ID`
7. `APP_AUTHZ_MODEL_ID`

Rauthy is local infrastructure, not a `web-bff` runtime dependency or code branch.

## Images

- **API**: Multi-stage build → `gcr.io/distroless/static-debian12:nonroot` (~6MB)
- **Web**: Multi-stage build → `ghcr.io/static-web-server/static-web-server:2` (Rust, ~10MB)
- **Gateway**: Multi-stage build → `gcr.io/distroless/static-debian12:nonroot` with Pingora + static-web-server (~15MB)

## Third-Party Image Pinning

Local template infrastructure uses explicit third-party image tags instead of `latest` where a registry tag can be verified. The current pins are intentionally evidence-based:

1. `openfga/openfga:v1.15.0` for the local authz reference adapter.
2. `nats:2.14.0-alpine` for the local JetStream broker.
3. `valkey/valkey:9.0.3-alpine` for the Redis-compatible cache.
4. `minio/minio:RELEASE.2025-09-07T16-13-09Z` and `minio/mc:RELEASE.2025-08-13T08-35-41Z` for object storage and bucket bootstrap.
5. `surrealdb/surrealdb:v3.0.5` for the optional app sidecar.
6. `ghcr.io/tursodatabase/libsql-server:e4beaca` for the optional sqld/libSQL server profile.
7. `public.ecr.aws/zinclabs/openobserve:v0.80.1` for the local observability UI/storage.
8. `otel/opentelemetry-collector-contrib:0.151.0` for the local OTLP collector.
9. `ghcr.io/vectordotdev/vector:0.55.0-alpine` for the optional logs profile.

Upgrade rule for this template:

1. Verify the candidate tag exists in the registry before editing YAML.
2. Prefer a stable release tag over `latest`; use commit-style tags only when the upstream image does not expose semantic tags.
3. Run compose config validation and at least one local smoke for any image used by the default local infrastructure lane.
4. Keep optional app, observability, and full-profile upgrades separate from core/auth upgrades when the upstream release span is large.
5. Do not delete compose volumes or global tool caches as a substitute for migration validation.

K3s base application images use explicit `:template-local` placeholders. They should be replaced by the build/release pipeline with a tag or digest, not by `latest`.

`Vector` is not part of the default observability startup. It is behind the `logs` compose profile because container log socket behavior differs between Linux Docker, Linux Podman, and macOS Podman machines. Rust tracing should point at the OTel Collector, not directly at Vector.
