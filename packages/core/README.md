# Core

Pure Rust domain ports and build optimization.

## Structure

```
core/
├── domain/         # Database port traits (LibSqlPort, SurrealDbPort) + TenantId re-export from kernel
└── workspace-hack/ # cargo-hakari unified dependency crate
```

## Design

### Domain Layer

Defines database access traits that services use as port boundaries:

```rust
pub trait LibSqlPort: Send + Sync {
    async fn health_check(&self) -> Result<(), DomainError>;
    async fn execute(&self, sql: &str, params: Vec<Value>) -> Result<u64, DomainError>;
    async fn query<T: DeserializeOwned>(&self, sql: &str, params: Vec<Value>) -> Result<Vec<T>, DomainError>;
}
```

### TenantId

`TenantId` is defined in `kernel` (with `Serialize`, `Deserialize`, `Display`) and re-exported here.
This ensures a **single canonical type** across the workspace.  Code using `domain::ports::TenantId`
and `kernel::TenantId` refers to the **same type**.

### Key Rule

**Ports depend on traits, not concrete implementations.** This means:
- Swap Turso → SurrealDB → Postgres by changing the adapter
- Service code is unchanged
- Full unit test coverage without a real database

### workspace-hack

Generated by `cargo-hakari` to unify transitive dependencies and speed up builds.
Do not edit manually. Run `cargo hakari generate` to regenerate.
