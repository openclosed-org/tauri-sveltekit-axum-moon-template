# Platform

> Platform capability traits — **当前零依赖，保留为未来使用**。

## 当前导出

| Trait | 职责 | 状态 |
|-------|------|------|
| `ConfigProvider` | 配置读取抽象 | ⚠️ 无消费者 |
| `TelemetryProvider` | 指标/日志抽象 | ⚠️ 无消费者 |
| `Clock` + `SystemClock` | 时间抽象 | ⚠️ 无消费者 |

## 为什么保留

这些 trait 为未来分布式场景下的平台能力抽象预留。当前阶段各服务和 worker
直接使用具体实现或更轻量的抽象（如 `kernel` 中的类型），不需要经过此层。

## 何时启用

当出现以下情况时启用：
1. 需要统一多环境配置读取接口
2. 需要跨服务/worker 的统一遥测协议
3. 需要可注入的时间抽象（测试时间依赖逻辑）
