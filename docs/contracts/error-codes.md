# Error Code Docs

> 状态：按需参考，不是默认上下文。
>
> 真理源：`packages/contracts/errors/**`。

## 1. 这份文档负责什么

这里只说明错误码文档的边界，不再手工维护完整错误码百科。

## 2. 当前真理源

错误码与错误响应形状应以以下内容为准：

1. `packages/contracts/errors/**`
2. 相关 handler / service 中的错误映射代码
3. 可执行 contract / type 检查

## 3. 为什么收口

旧版大而全错误码手册容易产生两个问题：

1. 代码里没有的错误码被文档写成已存在。
2. 错误语义、状态码映射、细节字段和真实实现逐渐漂移。

## 4. 当前建议用法

需要确认错误语义时：

1. 先看 contracts 中的 error 定义
2. 再看 server 层如何映射到 HTTP/Tauri/RPC 响应
3. 最后再把本文档当作导航页，而不是事实清单

## 5. 一句话结论

错误码的真理源在 contracts 和实际错误映射代码，不在手工维护的长篇文档。
