# 档位判定标准（Maturity Levels）

> 目的：为迭代计划、轨道切换与门禁强度提供统一判定基准。
> 规范：本文件为“SDD 规范驱动开发”的判定基础，后续所有计划文档须引用此档位定义。

---

## 1. 第一档（L1）：目录约定 + 弱约束

**特征**

- 主要靠目录与约定，规则未机器可执行或仅局部执行
- 真实入口与文档描述存在明显偏差
- 页面/模块可以直接绕过共识边界（无硬门禁）

**最低判定**

- 没有独立规则真相源
- 没有稳定的 gate 或只在局部启用
- 生成物与实现之间允许长期漂移

---

## 2. 第二档（L2）：结构真相源 + 局部门禁

**特征**

- 存在机器可读的真相源（codemap / schema / rules）
- 关键边界能被 validator 发现（存在性、依赖方向、基本结构）
- gate 已存在但以 warn 或局部 strict 为主

**最低判定**

- `validate-existence` / `validate-imports` 可运行
- 规则可解释且能阻止明显越界
- CI 分层开始收口，但未形成“生成物零漂移”闭环

---

## 3. 第三档（L3）：闭环平台化

**特征**

- 真相源唯一，模型可反向约束实现
- 生成物可删可再生，且有零漂移检查
- 变更影响可计算（impact 识别 + affected 执行）
- 合并门禁为编译式裁决，不依赖自觉

**最低判定**

- 生成物 drift 在 CI/合并前为 strict 阻塞
- 合约/模型变化必须通过 generator/validator 闭环
- 变更影响有明确可执行工具链（moon/affected + ruleset）

---

## 4. 当前仓库现实档位（事实基线）

- 后端：L2（结构真相源与局部门禁已落地，但尚未形成第三档闭环）
- 前端：L1（历史遗留结构明显，规则真相源未落地，门禁缺失）
- 集成：未启动（仅有规划，尚无实作 gate/CI 组合层）

> 注：以上为“事实基线”，不是目标态。所有轨道计划必须以事实基线为起点。
